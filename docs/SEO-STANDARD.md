## 1. Архітектура URL

### Основний домен (check-up.in.ua)

```
https://check-up.in.ua/ukr/[slug]
```

**Без проміжних рівнів** типу `/surgery/`, `/diagnoses/`, `/operations/`. Напрямкові сторінки (хірургія, гінекологія, кардіологія) не створюються. Кожен діагноз і кожна операція — окремий слаг першого рівня.

**Правильно:**
- `check-up.in.ua/ukr/mioma-matky`
- `check-up.in.ua/ukr/laparoskopichna-miomektomia`
- `check-up.in.ua/ukr/septoplastyka`

**Неправильно:**
- `check-up.in.ua/ukr/surgery/mioma-matky`
- `check-up.in.ua/ukr/gynecology/mioma-matky`

### Винятки — multi-level URL дозволені тільки для:

- **Чекапи (City Catalog):** `/ukr/checkup/female/[city]` — гео в URL виправдано існуючим SEO-трафіком
- **Careway модуль:** `/ukr/careway/simptomy/[slug]`, `/ukr/careway/rishennya/[slug]` — окрема контентна вертикаль
- **Каталог чекапів за категорією:** `/ukr/special-checkup/[slug]` — спеціальні програми

### Піддомени клінік

```
https://[clinic].check-up.in.ua/[city]/[slug]
```

- `[clinic]` — slug клініки (onclinic, rokl, medok)
- `[city]` — slug міста (kharkiv, kyiv, lviv, dnipro, rivne)
- `[slug]` — слаг діагнозу або операції

**Приклад:** `https://onclinic.check-up.in.ua/kharkiv/mioma-matky`

### Slug-правила

- Українська транслітерація без `-likuvannya` (мається на увазі)
- Без діакритики, без апострофів, тільки `[a-z0-9-]`
- Не дублювати між основним доменом і піддоменом одного діагнозу — це **різні URL** з **різним контентом**, не різні URL з однаковим контентом

### Canonical — обов'язкові правила

| Сторінка | Canonical |
|---|---|
| Основний домен Hub | На саму себе (`<link rel="canonical" href="https://check-up.in.ua/ukr/mioma-matky">`) |
| Основний домен Case | На саму себе |
| Піддомен клініки | На саму себе (НЕ на основний домен!) |
| Гео-сторінка чекапу | На саму себе (НЕ на загальну `/checkup/female`) |
| Загальна чекап-сторінка | На саму себе |

**Заборонено:** канонікал з піддомена на основний домен. Це сигнал Google "це дублікат" → піддомен видалиться з індексу через 2-6 місяців.

## 2. Карта інтентів і типів сторінок

Тип сторінки визначається тим, **що знає пацієнт і що шукає**. Це фундамент антиканібалізаційної стратегії.

| Тип | URL | Пацієнт знає | Шукає | Цільові запити | НЕ цільові запити |
|---|---|---|---|---|---|
| **1. Hub діагнозу (платформа)** | `/ukr/[diagnosis]` | Діагноз | Який метод обрати, де лікувати по Україні | "лікування міоми матки", "видалення міоми ціна", "методи лікування міоми", "як лікувати міому без операції" | Запити з гео-маркером (передаються Case клініки або Hub з гео-секцією) |
| **2. Case методу (платформа)** | `/ukr/[method]` | Метод обрано | Деталі методу, ціна в Україні, де роблять | "лапароскопічна міомектомія", "лапароскопічна міомектомія ціна", "як проходить лапароскопічна міомектомія" | Запити з гео-маркером і brand клініки |
| **3. Hub діагнозу (піддомен клініки)** | `[clinic]/[city]/[diagnosis]` | Діагноз + клініка | Що ця клініка пропонує | "міома Харків ON Clinic", "лікування міоми в ON Clinic" | Загальні без гео і без бренду |
| **4. Case методу (піддомен клініки)** | `[clinic]/[city]/[method]` | Метод + клініка | Деталі методу в цій клініці, ціна, запис | "лапароскопічна міомектомія Харків", "лапароскопія міоми ON Clinic", "лапароскопічна міомектомія Харків ціна" | Загальні без гео |
| **5. City Catalog (чекапи)** | `/ukr/checkup/[type]/[city]` | Тип чекапу + місто | Програми у місті з цінами | "жіночий чекап Київ", "чекап для жінок 40 років ціна Київ" | Запити без гео |
| **6. Symptom page (Careway)** | `/ukr/careway/simptomy/[symptom]` | Симптом, не діагноз | Що це може бути, куди йти | "рясні місячні причини", "що означає біль внизу живота у жінок" | Запити з конкретним діагнозом |
| **7. Decision page (Careway)** | `/ukr/careway/rishennya/[decision]` | Потребує орієнтації | Алгоритм рішення | "як обрати хірурга", "як підготуватися до операції", "що таке друга думка лікаря" | Конкретні діагнози і клініки |
| **8. Doctor profile** | `[clinic]/[city]/doctors/[slug]` | Ім'я лікаря | Досвід, ціна, запис | "лікар Афанасьєв Харків", brand-запити лікаря | Загальні без імені |

### Принцип розподілу intent

**Не-гео запит → платформа.** "лікування міоми матки" → Hub платформи.
**Гео-запит → піддомен клініки.** "лікування міоми Харків" → Hub клініки в Харкові.
**Brand-запит → піддомен клініки.** "ON Clinic міома" → Hub клініки.
**Метод без гео → Case платформи.** "лапароскопічна міомектомія" → Case платформи.
**Метод з гео → Case клініки.** "лапароскопічна міомектомія Харків" → Case клініки в Харкові.

### Коли в місті немає партнера

Якщо запит "лікування міоми Київ" але в Києві немає клініки-партнера:
- **НЕ створювати** окремий гео-Hub `check-up.in.ua/ukr/kyiv/mioma-matky` без партнера — буде канібалізація з основним Hub
- **Створити** гео-секцію в основному Hub (H2 "Лікування міоми в Києві" + текст + рекомендація найближчого міста)
- Коли з'явиться партнер у Києві — піддомен забирає на себе гео-запит, гео-секція в Hub скорочується або видаляється

## 3. Розподіл відповідальності: основний домен vs піддомен

Фундаментальне правило: **різний контент, різний обсяг, різна глибина**. Не дублювати.

### Hub діагнозу платформа vs Hub діагнозу клініка

| Блок | Платформа | Піддомен клініки |
|---|---|---|
| **Tl;dr-абзац (50-70 слів)** | ✓ Обов'язково | — |
| **H1 з містом** | — | ✓ Обов'язково |
| **Огляд всіх методів в Україні** | ✓ 6 методів | — |
| **Методи цієї клініки** | — | ✓ 2-3 методи що виконує клініка |
| **Квіз з 5 результатами (включно з ЕМА)** | ✓ Всі результати | ✓ Тільки ті результати, чиї методи доступні в клініці |
| **Цінові діапазони по Україні** | ✓ Таблиця по містах | — |
| **Конкретна ціна** | — | ✓ Одна цифра або вузький діапазон |
| **Матриця доступності методів по містах** | ✓ Унікальне для маркетплейсу | — |
| **Каталог клінік-партнерів** | ✓ Всі партнери з фільтрами | — |
| **Хірурги по Україні** | ✓ Grid з фільтром по містах і методах | Тільки хірурги цієї клініки |
| **Мікросекції "X чи Y"** | ✓ Обов'язково | — |
| **Підготовка + Реабілітація (детально)** | ✓ Детально | Коротко зі специфікою клініки |
| **Ризики, ускладнення, протипоказання** | ✓ Детально з джерелами | Згадка, не деталізація |
| **FAQ розгорнутий (10-15 питань)** | ✓ Всі питання | 4-5 клінік-специфічних питань |
| **Чек-лист "як обрати клініку"** | ✓ Унікально для платформи | — (клініка не може чесно цього робити) |
| **Перевірено редакцією платформи** | ✓ Незалежний рев'юер | ✓ Лікар клініки |
| **Запис на консультацію** | CTA на каталог клінік | ✓ Прямий CTA з телефоном клініки |
| **Загальний обсяг** | 2500-4000 слів | 800-1500 слів |

### Case методу платформа vs Case методу клініка

| Блок | Платформа | Піддомен клініки |
|---|---|---|
| **Опис методу детально** | ✓ 300-500 слів | ~100 слів про специфіку клініки |
| **Як проходить операція (етапи)** | ✓ Деталізовано | Коротко |
| **Цінові діапазони по містах** | ✓ Таблиця | — |
| **Конкретна ціна клініки** | — | ✓ Одна цифра з sidebar |
| **Порівняння з іншими методами** | ✓ Таблиця | Згадка з лінком на Hub |
| **Каталог клінік які виконують** | ✓ Всі партнери | — |
| **Підготовка + Реабілітація** | ✓ Детально, освітньо | ~ Скорочено |
| **Ризики і ускладнення** | ✓ Детально з відсотками | ~ Згадка |
| **FAQ** | ✓ 10-15 питань | 4-5 клінік-питань |
| **Дослідження, цитати з джерел** | ✓ Унікально | — |
| **Підсилений Schema** | ✓ MedicalProcedure без provider | ✓ MedicalProcedure з provider + MedicalBusiness |

### Як не дублювати контент

1. **Опис методу слово-в-слово копіювати заборонено.** На піддомені переписати або скоротити на 50%+.
2. **FAQ-блок** — на піддомені тільки клініка-специфічні питання (запис, оплата, графік), не загальні (загальні залишаються на Hub платформи).
3. **Title/H1** — піддомен **обов'язково** з містом і брендом клініки. Hub платформи — без міста.
4. **Внутрішня перелінковка**: Hub→піддомен (так, для кожного офера), піддомен→Hub того самого методу (НІ — це сигнал що Hub є вищим за піддомен).

## 4. Title / H1 / Meta description — формули

### Загальні правила

- **Title:** 60 символів видимих, до 110 символів індексованих (розширений title дає Google варіативність)
- **H1:** один на сторінці, унікальний на платформі, відрізняється від title (не дублювати)
- **Meta description:** 150-160 символів, обов'язково з ціною/числом якщо доречно (підвищує CTR)
- **Без emoji в title/H1/meta**
- **Без CAPS LOCK блоків**
- **Місто завжди в title, якщо це піддомен клініки або гео-сторінка**

### Шаблони по типах

#### Тип 1 — Hub діагнозу (платформа)

```
Title:       [Назва дії] [діагнозу]: [3-4 ключі через кому] | check-up.in.ua
H1:          [Назва дії] [діагнозу]: [розширення без бренду]
Meta:        [Скільки методів]. [Опис підбору]. [Гео-маркер міст-партнерів].
```

Приклад:
```
Title:       Видалення міоми матки: операції, ціна, вибір методу — лапароскопія, гістероскопія, ЕМА | check-up.in.ua
H1:          Видалення міоми матки: операції в Україні та вибір методу
Meta:        6 методів видалення міоми матки: лапароскопія, гістероскопія, ЕМА, гістеректомія. Пройдіть квіз — оберіть метод і клініку в Києві, Харкові, Львові, Дніпрі.
```

#### Тип 2 — Case методу (платформа)

```
Title:       [Метод]: ціна, як проходить, де роблять в Україні | check-up.in.ua
H1:          [Метод]: огляд процедури та клініки в Україні
Meta:        [Що таке метод у 1 реченні]. Вартість в Україні від X до Y грн. Клініки в [3-5 міст].
```

Приклад:
```
Title:       Лапароскопічна міомектомія: ціна в Україні, як проходить, відновлення | check-up.in.ua
H1:          Лапароскопічна міомектомія: метод видалення міоми зі збереженням матки
Meta:        Лапароскопічна міомектомія — видалення міоми через 3-4 проколи зі збереженням матки. Ціна в Україні від 28 000 до 65 000 грн. Клініки в Києві, Харкові, Львові.
```

#### Тип 3 — Hub діагнозу (піддомен клініки)

```
Title:       [Дія] [діагнозу] в [Місто] — [Бренд клініки]: [методи через кому]
H1:          [Дія] [діагнозу] в [Місто] (або: Лікування [діагнозу] в [Бренд клініки])
Meta:        [Бренд клініки, Місто] виконує [методи]. Ціни від X грн. Хірурги: [регалії]. Запис онлайн.
```

Приклад:
```
Title:       Лікування міоми матки в Харкові — ON Clinic: лапароскопія, гістероскопія, гістеректомія
H1:          Лікування міоми матки в ON Clinic Харків
Meta:        ON Clinic Харків виконує лапароскопічну та гістероскопічну міомектомію. Ціни від 18 000 грн. Хірурги — к.м.н., 24-34 роки досвіду. Запис онлайн.
```

#### Тип 4 — Case методу (піддомен клініки)

```
Title:       [Метод] в [Місто] — [Бренд клініки], від [Ціна] грн
H1:          [Метод] в [Місто] (підзаголовок під H1 з брендом клініки)
Meta:        [Метод] в [Бренд клініки Місто]. Вартість від X грн, включає [що входить]. Хірурги [регалії]. Обладнання [бренд].
```

Приклад:
```
Title:       Лапароскопічна гістеректомія в Харкові — ON Clinic, від 28 895 грн
H1:          Лапароскопічна гістеректомія в Харкові
Meta:        Екстирпація матки лапароскопічним методом в ON Clinic Харків. Вартість від 28 895 грн, включає операцію, анестезію, стаціонар. Хірурги к.м.н., вища категорія.
```

#### Тип 5 — City Catalog (чекапи)

```
Title:       [Чекап] в [Місто]: програми, ціни, клініки | check-up.in.ua
H1:          [Чекап] в [Місто]
Meta:        Програми [чекапу] в [Місто] — ціни від X грн. Клініки-партнери: [N]. Підберіть програму під вік і ризики через квіз.
```

#### Тип 6 — Symptom page

```
Title:       [Симптом] — причини, коли до лікаря, обстеження | check-up.in.ua
H1:          [Симптом]: що це може означати
Meta:        [Симптом] — можливі причини: [3 діагнози]. Коли терміново до лікаря. Які обстеження пройти.
```

#### Тип 7 — Decision page

```
Title:       [Питання]: [короткий опис рішення] | check-up.in.ua
H1:          [Питання]
Meta:        [Чек-лист або 3-5 кроків опис]. Що врахувати, на що звернути увагу.
```

#### Тип 8 — Doctor profile

```
Title:       [ПІБ лікаря] — [Спеціальність], [Місто, Клініка], запис на консультацію
H1:          [ПІБ лікаря]
Meta:        [Спеціальність, регалії]. [Років досвіду]. [Спеціалізація]. Ціна консультації від X грн. Запис онлайн.
```

## 5. Schema.org розмітка по типах сторінок

### Базові правила

- JSON-LD у `<head>` (не Microdata, не RDFa)
- Валідується через Google Rich Results Test перед мержем
- Несе тільки реальні дані — фейкові поля заборонені (особливо `aggregateRating` без реальних відгуків)

### Розмітка по типах

#### Тип 1 — Hub діагнозу (платформа)

```json
{
  "@context": "https://schema.org",
  "@type": "MedicalWebPage",
  "name": "Видалення міоми матки",
  "url": "https://check-up.in.ua/ukr/mioma-matky",
  "mainEntity": {
    "@type": "MedicalCondition",
    "name": "Міома матки",
    "alternateName": ["Лейоміома матки", "Фіброміома"],
    "code": {
      "@type": "MedicalCode",
      "codingSystem": "ICD-10",
      "codeValue": "D25"
```

## 6. GEO-блок — стандарт

GEO-блок — обов'язковий для типів 1, 2, 3, 4, 5. Це найважливіший SEO-блок для AI Overview і Local SERP.

### Технічні вимоги

- **Статичний HTML.** Не всередині `<script>`, не в JS-tabs, не в accordion з `display:none`.
- **Видимий через View Source** (Ctrl+U) на завантаженій сторінці.
- Якщо контент згорнутий — використовувати `data-collapsed` + `max-height: 0`, не `display:none`.
- Named entities (міста, клініки, лікарі, телефони) видимі без виконання JavaScript.

### Структура для Hub платформи (тип 1)

```html
<section class="geo-block">
  <h2>Лікування міоми матки в Україні</h2>
  <p>Платформа check-up.in.ua об'єднує клініки які виконують операції з міоми матки у [N міст]: Київ, Харків, Львів, Дніпро, Рівне. У базі — [N] хірургів з досвідом від 15 до 34 років.</p>
  
  <ul>
    <li><strong>Київ:</strong> [клініка], від [X] грн, [метод]</li>
    <li><strong>Харків:</strong> ON Clinic, від 18 000 грн, лапароскопічна та гістероскопічна міомектомія</li>
    <li><strong>Рівне:</strong> РОКЛ, від 16 000 грн, повний спектр методів</li>
  </ul>
</section>
```

### Структура для піддомена клініки (тип 3-4)

```html
<section class="geo-block">
  <h2>Лікування міоми матки в ON Clinic Харків</h2>
  <p>ON Clinic Левада, м. Харків, пр. Героїв Харкова, 257. Гінекологи виконують лапароскопічну та гістероскопічну міомектомію з 2006 року. Хірурги Афанасьєв І.В. (34 роки досвіду, к.м.н.) та Стрюков Д.В. (24 роки досвіду, к.м.н.). Запис: телефон [номер], Telegram [контакт].</p>
</section>
```

### Що НЕ можна

- Стокові тексти типу "ми працюємо по всій Україні"
- GEO-блок як футер (footer не індексується як content)
- GEO-блок в JS-табах ("Київ | Харків | Львів" — інтерактивно, без HTML)
- GEO-блок в accordion з `display:none` за замовчуванням

## 7. E-E-A-T блок — стандарт

E-E-A-T (Experience, Expertise, Authoritativeness, Trust) — критичний для медичних сайтів (YMYL). Google віддає перевагу сторінкам з видимими сигналами довіри.

### Обов'язкові елементи на кожній медичній сторінці

```html
<section class="eeat-block">
  <div class="reviewer">
    <strong>Перевірено лікарем:</strong> [ПІБ, спеціальність, регалії]
    <span class="reviewer-credentials">[Категорія, ступінь, місце роботи]</span>
  </div>
  
  <div class="dates">
    <span>Опубліковано: [дата]</span>
    <span>Оновлено: [дата]</span>
  </div>
  
  <div class="sources">
    <strong>Джерела:</strong>
    <ul>
      <li><a href="https://dec.gov.ua/mtd/...">УКПМД "Лейоміома матки" (наказ МОЗ України № 147/2023)</a></li>
      <li>Mayo Clinic Family Health Book, розділ 33 "Women's Health"</li>
    </ul>
  </div>
  
  <div class="conflict-disclosure">
    <small>check-up.in.ua — медичний маркетплейс. Ми отримуємо комісію від клінік-партнерів. Це не впливає на медичний зміст: всі дані перевірені незалежним рев'юером.</small>
  </div>
</section>
```

### Хто може бути рев'юером

- Лікар з відповідною спеціальністю і регаліями (категорія, ступінь)
- ПІБ обов'язково реальне, не "редакція сайту"
- Видима кваліфікація — місце роботи, регалії
- На піддомені клініки рев'юер = лікар цієї клініки
- На основному домені рев'юер = незалежний медичний редактор/консультант платформи

### Що заборонено

- **Не вигадувати імена рев'юерів.** Якщо немає реального — використовувати `[УТОЧНИТИ]` і повідомити стейкхолдеру.
- **Не приписувати чужі регалії.**
- **Не використовувати "перевірено редакцією" без імені** — це нульовий E-E-A-T сигнал.

### Disclosure фінансових відносин

Платформа отримує комісію від клінік. Це треба явно розкривати — це **підвищує** довіру за Google YMYL guidelines, не знижує.

## 8. Внутрішня перелінковка

### Дозволені переходи

| З сторінки | На сторінку | Дозволено? |
|---|---|---|
| Hub діагнозу (платформа) | Case методу (платформа) | ✓ Так, обов'язково |
| Hub діагнозу (платформа) | Hub діагнозу (піддомен клініки) | ✓ Так, для кожного офера |
| Hub діагнозу (платформа) | Case методу (піддомен клініки) | ✓ Так, з блоку оферів |
| Case методу (платформа) | Hub діагнозу (платформа) | ✓ Так, breadcrumbs і "повернутись до огляду" |
| Case методу (платформа) | Case методу (піддомен клініки) | ✓ Так, з каталогу клінік |
| Hub клініки | Case клініки тієї ж клініки | ✓ Так |
| Case клініки | Інший Case тієї ж клініки | ✓ Так, "альтернативні методи" |
| Case клініки | Doctor profile тієї ж клініки | ✓ Так |
| **Піддомен клініки** | **Той самий діагноз/метод на основному домені** | **✗ НІ — це канібалізаційний сигнал** |
| **Піддомен клініки A** | **Піддомен клініки B (конкурент)** | **✗ НІ — клініки не повинні рекламувати один одного** |
| Hub діагнозу (платформа) | Symptom page (Careway) | ✓ Так, в FAQ або "коли терміново" |
| Symptom page | Hub діагнозу | ✓ Так, "наступний крок" |
| Будь-яка медична сторінка | Decision page | ✓ Так, "як обрати клініку/лікаря" |

### Заборонені

- **Cross-link з піддомена клініки на головну платформи з якорем на той же діагноз.** Якщо піддомен ON Clinic Харків лікує міому — лінк на `check-up.in.ua/ukr/mioma-matky` створює сигнал "це підрозділ Hub" і знижує ранжування піддомена.
- **Cross-link з піддомена клініки на конкурентний піддомен.** Адонис не повинен лінкувати на ON Clinic.
- **Cyclic linking між Hub і Case того самого діагнозу через кожну згадку.** Достатньо breadcrumbs + 1-2 контентних посилань.

### Якщо піддомен хоче послатись на платформу

Дозволено посилання на **головну платформи** (`check-up.in.ua`), не на конкретний Hub. Або на Careway/Decision page (загальна корисна інформація, не конкурує з піддоменом).

## 9. Канібалізаційний чек-лист

Виконати **перед мержем кожної сторінки в main**.

### Етап 1 — Архітектурна перевірка

- [ ] URL відповідає схемі з розділу 1
- [ ] Canonical вказує на саму себе, не на іншу сторінку
- [ ] Тип сторінки з розділу 2 чітко визначений
- [ ] Цільові запити з розділу 2 збігаються з фактичним H1/title
- [ ] НЕ цільові запити не таргетимо в title/H1

### Етап 2 — Перевірка дублювання з іншими сторінками платформи

- [ ] H1 цієї сторінки **не повторює** H1 жодної іншої сторінки на платформі (включно з піддоменами)
- [ ] Title унікальний на платформі
- [ ] Контент (перші 300 слів) не дублює іншу сторінку платформи
- [ ] Якщо є Hub платформи і Case клініки на ту ж тему — розподіл блоків з розділу 3 виконано

### Етап 3 — Hub платформа vs Hub клініки

Якщо існує і Hub платформи, і Hub клініки на той самий діагноз:
- [ ] Hub платформи НЕ містить міста в title/H1
- [ ] Hub клініки містить місто і бренд в title/H1
- [ ] Hub платформи показує **всі методи в Україні**, не тільки методи цієї клініки
- [ ] Hub клініки показує **тільки методи цієї клініки**, без матриці інших міст
- [ ] У Hub платформи зі сторінки клініки немає reverse-лінка на Hub платформи з якорем на той же діагноз

### Етап 4 — Schema і верифікація

- [ ] Schema валідна (Google Rich Results Test пройдено)
- [ ] MedicalProcedure має правильний `procedureType` і `bodyLocation`
- [ ] На піддомені клініки є `MedicalBusiness` з реальною адресою
- [ ] `reviewedBy` має реальне ПІБ або позначку `[УТОЧНИТИ]`
- [ ] Немає фейкового `aggregateRating`

### Етап 5 — GEO-блок і E-E-A-T

- [ ] GEO-блок видимий через View Source (без JavaScript)
- [ ] GEO-блок не в accordion з `display:none`
- [ ] GEO-блок містить named entities (міста, клініки, лікарі, телефони)
- [ ] E-E-A-T блок присутній з реальним або `[УТОЧНИТИ]` рев'юером
- [ ] Дати опублікування і оновлення видимі
- [ ] Джерела з гіперпосиланнями (МОЗ, Mayo Clinic)

### Етап 6 — Внутрішня перелінковка

- [ ] З сторінки немає заборонених лінків (див. розділ 8)
- [ ] Lin на конкретні клініки-партнери ведуть на їх піддомени, не на гіпотетичні URL основного домена
- [ ] Sitemap.xml оновлений з новим URL

### Етап 7 — Мобільна перевірка

- [ ] H1 видимий вище fold на мобільному (380px viewport)
- [ ] Title не обрізаний на 60 символах в SERP-preview
- [ ] Meta description читабельна
- [ ] GEO-блок не прихований за accordion на мобільному

### Якщо хоч один пункт NO — НЕ мерж

Cowork зупиняє роботу і повідомляє стейкхолдеру конкретний пункт що не виконується.

## 10. AI Overview / GEO оптимізація

GEO (Generative Engine Optimization) — окремий шар поверх класичного SEO. Орієнтація на цитованість в AI Overview, ChatGPT з web search, Perplexity, Bing Copilot.

### Tl;dr-абзац (обов'язково для типів 1, 2)

Перший абзац після H1, 50-70 слів, прямо відповідає на головне питання. Цей абзац AI Overview цитує дослівно.

Структура:
- 1 речення з відповіддю по суті
- 1-2 речення з основними числами або переліком
- 1 речення з висновком "що залежить від чого"

### FAQ-блок з Schema FAQPage (обов'язково)

10-15 питань. **Кожна відповідь самодостатня, 2-3 речення, з цифрами.** Без посилань "як ми вже згадували вище".

Формат який LLM цитують:

```
Питання: Чи зможу я завагітніти після лапароскопічної міомектомії?
Відповідь: Так. Лапароскопічна міомектомія зберігає матку, після операції фертильність відновлюється. Планувати вагітність можна через 6-12 місяців.
```

### Конкретика з одиницями

LLM цитують факти з цифрами і одиницями: "до 10 см", "70-80% жінок до 50 років", "відновлення 7-14 днів", "вартість 35 000-90 000 грн". Без діапазонів — не цитується.

### Порівняльні таблиці в HTML

Не зображенням, не SVG. Чистий `<table>` з `<thead>`/`<tbody>`. Perplexity і ChatGPT парсять HTML-таблиці і часто показують дослівно.

### Мікросекції "X чи Y"

Прямі порівняння двох варіантів — захоплюють LLM-запити типу "що краще лапароскопія чи гістероскопія":

```html
<h3>Лапароскопія чи гістероскопія для міоми — коли який метод</h3>
<p>[3-4 речення прямою мовою з ключовими відмінностями]</p>
```

### Видима дата оновлення

`Оновлено: [YYYY-MM-DD]` на сторінці. LLM пріоритизують свіжий контент.

### Канонічна назва + синоніми у першому H2

Перший H2 містить основну назву і синоніми: "Що таке міома матки (лейоміома, фіброміома)". Це дає LLM зрозуміти що це синоніми → процитують вашу сторінку на будь-який варіант запиту.

### Авторство видиме

E-E-A-T блок (розділ 7) + Schema `reviewedBy` працюють разом: AI Overview бачить що сторінка має автора-експерта.

### Що НЕ робити

- Не ховати корисний контент в JS-табах — LLM не виконують JavaScript при cravling
- Не використовувати CSS `text-shadow`, transformації для важливого тексту
- Не дублювати tl;dr тричі по сторінці — один раз, на початку
